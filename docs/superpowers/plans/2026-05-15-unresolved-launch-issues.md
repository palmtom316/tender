# 2026-05-15 尚未解决问题跟踪计划

## 背景

2026-05-15 使用 tender 系统现有流程、DeepSeek 生成“2026 年国网重庆市区供电分公司年度 10kV 配网工程年度框架招标”第 8 章 DOCX 后，暴露出上线验收风险：

- 生成耗时约 140 分钟。
- 导出的第 8 章 DOCX 未达到目标 100 页。
- 内容不完整，不能证明覆盖全部 8.1-8.15 小节、全部图表、全部招标约束。
- 已修复模板 stale gate 代码审查中的阻塞问题，但长篇标书生成质量闭环仍未完成。

本文档只跟踪尚未解决的问题；已提交修复项不再重复展开。

## 上线结论

当前系统可以作为“模板编辑、生成、导出链路可跑通”的 MVP 基础，但不能作为“可稳定生成完整 100 页技术标章节”的生产承诺上线。

以下 P0 项未关闭前，不应对外承诺：

- “第 8 章可按目标页数生成 100 页”。
- “系统可自动保证长篇标书内容完整”。
- “全部图表和正文均已通过自动审查”。

## P0-1 目标页数没有硬闸门

**问题**

当前 `target_pages=100` 只是传入生成上下文/提示词的软约束。系统没有自动页数估算、DOCX 渲染后页数统计、低于目标页数的拒绝/续写机制。

**影响**

模型返回短文时，系统仍会进入导出流程，导致用户拿到明显不足页数的 DOCX。

**待实施任务**

- [ ] 在章节生成结果中记录 `target_pages`、估算页数、估算依据和生成轮次。
- [ ] 增加章节级页数估算器：基于中文字符数、标题层级、表格、图片、分页符估算页数。
- [ ] 增加 DOCX 导出后页数检测能力；优先使用可自动化的 LibreOffice/PDF 转换页数统计，容器不可用时明确降级为未校验。
- [ ] 在导出闸门中加入 `page_count_gate`：低于目标下限时阻止“最终版导出”，允许“草稿导出”。
- [ ] 前端导出页展示目标页数、估算页数、实际页数、是否达标。

**验收标准**

- [ ] 目标 100 页时，实际页数低于 90 页会阻止最终导出。
- [ ] 页数未校验时，导出闸门显示“格式/页数未校验”，不能静默通过。
- [ ] 单测覆盖：达标、不达标、无法检测三种状态。
- [ ] E2E 覆盖：第 8 章目标 100 页生成后，导出闸门可显示页数证据。

## P0-2 长篇章节仍是单次生成，缺少分段续写

**问题**

第 8 章 8.1-8.15 是长篇章节，目前生成方式仍偏单次请求/单草稿保存。DeepSeek 单次输出受 token 上限、上下文压缩和模型遵循度影响，无法稳定生成 100 页。

**影响**

即使提示词要求 100 页，模型也可能只返回十几页内容；重跑会继续浪费时间和 token。

**待实施任务**

- [ ] 将第 8 章拆成 8.1-8.15 子章节生成任务，每个子章节独立设置目标页数、输入约束和图表清单。
- [ ] 增加 continuation loop：单节未达到最低字符/页数时，基于已生成内容继续补写，直到达标或达到最大轮次。
- [ ] 增加章节总装：按模板目录顺序合并子章节草稿、图表、表格、分页符和页眉页脚。
- [ ] 增加失败恢复：某一小节失败时可重跑该小节，不重跑全章。
- [ ] 记录每一小节的 provider/model、prompt hash、输入 token、输出 token、耗时、续写轮次。

**验收标准**

- [ ] 8.1-8.15 每节都有独立生成记录。
- [ ] 单节未达标会自动续写或明确标记失败。
- [ ] 失败重跑只影响对应小节。
- [ ] 第 8 章总装 DOCX 能追溯到每个小节草稿和图表资产。

## P0-3 内容完整性没有自动审查

**问题**

当前 self-check 只能做粗粒度检查，例如策略小节数量、图表占位符数量、是否含报价词。它不能证明：

- 8.1-8.15 全部出现且内容充足。
- 每个小节覆盖对应招标约束、技术规范、评分点。
- 固定文本、表格、资产占位符、AI 提示词、AI 图表代码、页面格式均进入最终 DOCX。

**影响**

生成文档可能“看起来有内容”，但漏掉关键评审项，影响投标文件可用性。

**待实施任务**

- [ ] 建立第 8 章内容清单：8.1-8.15 的必备标题、最小字数、必备图表、必备表格、必备响应点。
- [ ] 将招标文件解析出的技术约束映射到 8.1-8.15，每条约束要求有响应位置。
- [ ] 生成后执行 coverage check：按小节检查标题、字数、约束覆盖、图表引用、资产占位符、分页符。
- [ ] 将 coverage check 接入导出闸门；P0 缺口阻止最终导出。
- [ ] 前端展示缺口列表，并支持跳转到对应模板块/章节草稿。

**验收标准**

- [ ] 任一 8.x 小节缺失时导出闸门阻止最终导出。
- [ ] 任一必备图表缺失时导出闸门阻止最终导出。
- [ ] 招标硬约束无响应位置时导出闸门阻止最终导出。
- [ ] coverage report 可导出为审查附件。

## P0-4 图表闭环仍需验收

**问题**

本次生成元数据中出现多个图表 key，但 DOCX media 数量少于预期。需要确认是图表复用、未引用、未渲染、未插入，还是导出统计方式不准确。

**待实施任务**

- [ ] 生成图表资产清单：placeholder_key、chart_type、status、rendered_png_path、approved 状态。
- [ ] 对比正文 `{{chart:*}}` 引用、`chart_asset` 表、DOCX media 三者。
- [ ] 增加导出前图表闸门：正文引用的图表必须存在、已渲染、已审核或允许草稿插入。
- [ ] 增加导出后图表验证：DOCX 中不能残留 `{{chart:*}}`，并记录插入成功数量。
- [ ] 前端导出页展示“引用图表 / 已生成 / 已插入 / 未通过”。

**验收标准**

- [ ] 所有正文引用图表都有资产记录。
- [ ] 所有必备图表都有 rendered PNG/SVG。
- [ ] 最终 DOCX 无 `{{chart:*}}` 残留。
- [ ] 图表插入数量与引用清单一致，允许复用时需要在报告中说明。

## P1-1 生成耗时过长且缺少阶段化进度

**问题**

单次测试耗时约 140 分钟，用户无法判断卡在哪个阶段，也无法只重跑失败片段。

**待实施任务**

- [ ] 将生成流程拆分为：准备上下文、生成小节、生成图表、审查、总装、导出。
- [ ] 每阶段写入任务进度、耗时、错误、重试次数。
- [ ] 前端展示阶段化进度和失败原因。
- [ ] 支持取消任务、重试失败阶段、重试单小节。

**验收标准**

- [ ] 用户能看到当前阶段和预计剩余工作。
- [ ] 任一阶段失败不会丢失已完成小节。
- [ ] 失败重试不需要重新跑完整 140 分钟流程。

## P1-2 DeepSeek 模型/参数策略未产品化

**问题**

当前可证明使用了 DeepSeek，但模型选择、max_tokens、续写策略、失败降级策略尚未产品化。长篇生成对模型和参数非常敏感。

**待实施任务**

- [ ] 将章节生成模型配置落到项目/模板级配置。
- [ ] 区分“快速草稿模型”和“最终长文模型”。
- [ ] 为长文生成配置最大输出、温度、续写轮次、每节目标字数。
- [ ] 记录每次生成使用的完整模型参数，写入 generation run metadata。

**验收标准**

- [ ] 每个生成结果可追溯 provider/model/参数。
- [ ] 不同项目类型可选择不同默认模型策略。
- [ ] 长篇最终生成不默认使用短输出策略。

## P1-3 模板编辑预览仍不等同于真实 DOCX 精排

**问题**

模板编辑右侧预览是结构化预览，不是 Word 引擎级真实分页。页眉、页脚、分页符、表格跨页、图片尺寸会在最终 DOCX 中产生差异。

**待实施任务**

- [ ] 明确区分“结构预览”和“DOCX 渲染预览”。
- [ ] 增加真实 DOCX 预览任务：保存模板后可生成预览 DOCX/PDF。
- [ ] 将页眉、页脚、分页符、节分隔符、页面方向纳入预览校验。
- [ ] 前端展示预览生成时间和模板 revision_no。

**验收标准**

- [ ] 用户能看到当前预览对应的模板 revision。
- [ ] 结构预览不再被文案暗示为最终精排。
- [ ] 真实 DOCX/PDF 预览能展示页眉页脚和分页效果。

## P1-4 真实招标文件解析到模板目录的差异调整仍需闭环

**问题**

流程设计要求：解析招标文件目录与原模板目录不一致时，在模板调整区调整。但目录差异调整后的生成影响、覆盖验证、版本追踪还需完整验收。

**待实施任务**

- [ ] 对目录 reconciliation 结果生成可操作任务：新增、删除、重命名、移动、合并章节。
- [ ] 目录调整后刷新模板 revision，并触发受影响草稿/图表 stale。
- [ ] coverage check 使用调整后的项目模板目录，而不是固定基线目录。
- [ ] 前端展示“招标目录差异已处理/未处理”状态。

**验收标准**

- [ ] 未处理 critical 目录差异时不能进入最终生成。
- [ ] 目录调整后生成结果使用调整后的目录。
- [ ] 调整记录可追溯到模板 revision。

## P2-1 上线验收样本不足

**问题**

目前只有单个第 8 章长文样例暴露问题，尚不足以说明系统对不同项目类型稳定。

**待实施任务**

- [ ] 建立至少 3 个真实招标文件验收样本：配网施工、监理/设计、商务标资产型章节。
- [ ] 每个样本跑完整流程：建项目、选模板、目录调整、模板编辑、生成、图表、导出、审查。
- [ ] 输出验收报告：页数、覆盖率、图表、耗时、人工返工点。

**验收标准**

- [ ] 每个样本都有可复现的运行记录和导出物。
- [ ] P0 问题全部关闭后再允许上线验收。

## 建议实施顺序

1. P0-2 分段生成与续写。
2. P0-1 页数硬闸门。
3. P0-3 内容完整性审查。
4. P0-4 图表闭环验收。
5. P1-1 阶段化进度与失败重试。
6. P1-2 DeepSeek 参数策略。
7. P1-3 真实 DOCX/PDF 预览。
8. P1-4 目录差异调整闭环。
9. P2-1 多样本验收。
