## 0. 产品定位确认

经业务方澄清，tender 当前定位与边界如下，本审核报告均按此定位修订：

- **使用对象:** 电力工程施工单位
- **首期支持业务线（4 类）:**
  1. 变电劳务及专业分包
  2. 10kV 项目（配电网建设/改造）
  3. 电力运维项目（变电/线路/配电运维）
  4. 用户高低压供配电工程
- **不在首期范围内（明确不做、保留扩展接口）:**
  - 经济标 / 投标报价编制（含工程量清单计价、综合单价分析、控制价红线校验等）
  - 房建、市政、公路、铁路、水利等其他工程类别
  - 电子保函、造价软件深度集成、招投标平台直连
- **未来扩展方向:** 通用化标书编制软件（含报价、其他工程类别、其他评标办法）

> 这意味着 tender 的核心交付是 **"非报价部分"** 的标书：资格商务（资质/业绩/人员）+ 技术标 + 配套图表 + 排版交付，并能与外部报价文件**整合打包**。下面所有意见均围绕这一边界展开。

## 1. 总体评价

两份文档在产品架构层面是清晰的：约束—结构—内容三层分离、模板与招标融合、确认闸门、章节追溯链路、版本化。从软件工程角度方向正确。

但作为电力施工投标实务来检视，存在以下需补强的方面：

1. **业务线维度未显式建模**：4 类业务线（变电分包/10kV/运维/用户配电）的章节、资质、人员、业绩、评分点差异巨大，不是一套模板能覆盖。
2. **投标合规要件未独立成层**：实质性响应、废标条款、符合性检查、签章/正副本/密封是电力施工废标高发区，需独立闸门。
3. **流程生命周期不完整**：流程停在 "delivered + archived"，丢失了"递交—开标—评标—中标公示—合同签订"段，影响后续复盘数据完整性。
4. **电力施工特有要素缺位**：特种作业证、工作票/操作票、电压等级业绩匹配、变电/线路/配电特有图表等。
5. **扩展接口未显式预留**：经济标、其他工程类别等明确不做，但应在数据模型、模板包类型、约束分类、服务接口上预留扩展点，否则未来通用化代价大。

## 2. 产品定位与扩展接口建议

### 2.1 spec 应新增 §0「适用范围与产品边界」

文档开头补一段，明确：

- 当前覆盖业务线（4 类）
- 当前**不覆盖**的能力（经济标、其他工程类别、平台直连等）
- 与外部系统的衔接边界（报价文件由外部出具后整体打包）
- 预留升级方向

避免后续团队/外部读者把它当作通用招投标系统理解。

### 2.2 数据模型显式预留扩展点

| 维度 | 当前文档 | 建议预留 |
|---|---|---|
| 投标分册 | 商务标、技术标、报价标、资格审查册 | 报价标分册保留**占位**（不实现内容生成，仅支持外部文件挂载） |
| 模板包类型 | 按 `project_type` 区分 | `project_type` 改为二级：`industry`(电力/房建/市政...) + `sub_type`(变电分包/10kV/运维/用户配电...)，首期只填电力 |
| 约束分类 | 业务标、技术标、评分、废标... | 增加 `pricing_constraint` 占位（首期不消费，但解析阶段可识别并存储） |
| 章节生成策略 | data_insert / template_render / ai_summary / manual_required | 增加 `external_attached`（外挂报价、外挂图纸等不参与 AI 生成的内容） |

### 2.3 服务层接口预留

`BidPricingService`、`PlatformIntegrationService`（招投标平台对接）等暂不实现，但建议在 plan 的 File Map 中作为**注释占位**写明，让架构边界从一开始就清晰。

## 3. 电力施工投标实务专项补强（重点）

### 3.1 项目立项字段（spec §2.1 / plan Phase 1）

当前字段不足，建议补：

**通用合规字段:**
- `evaluation_method`：评标办法（综合评估法 / 经评审的最低投标价法 / 合理低价法）— 影响模板与评分映射
- `qualification_review_type`：资格审查方式（资格预审 / 资格后审）
- `submission_deadline`：递交截止时间（取代 `tender_deadline` 的歧义）
- `bid_opening_time`：开标时间（与递交常常不同）
- `bid_validity_period`：投标有效期（影响资质证书有效期校验）
- `bid_bond_amount` / `bid_bond_form` / `bid_bond_deadline`：投标保证金金额、形式（电汇/保函）、到账截止
- `tender_platform`：招投标平台标识（国网 ECP / 南网平台 / 省级公共资源交易平台 / 线下）
- `is_consortium_allowed`：是否允许联合体投标
- `is_subcontract_allowed`：是否允许分包

**电力业务字段:**
- `business_line`：业务线（4 选 1：变电劳务及专业分包 / 10kV 项目 / 电力运维 / 用户高低压供配电）
- `voltage_level`：电压等级（500kV / 220kV / 110kV / 35kV / 10kV / 0.4kV，多选）
- `project_scope`：项目范围（设计 / 施工 / 试验 / 调试 / 运维，多选）
- `employer_type`：发包人类型（国网 / 南网 / 地方电力公司 / 工矿企业 / 园区物业 / 总承包方）
- `is_live_work_required`：是否涉及带电作业
- `controlled_price`：招标控制价（**仅作为元数据存储与展示，不参与报价编制**，便于风险提示）

**工作流状态扩展（spec §2.1）:**

```
created
→ source_uploaded
→ analysis_running
→ constraints_pending_confirmation
→ outline_pending_confirmation
→ drafting
→ draft_reviewing
→ revision_pending_confirmation
→ compliance_check       # 新增：符合性自查（废标风险扫描）
→ final_layout
→ external_pricing_merge # 新增：外部报价文件挂载（占位，可跳过）
→ bid_sealed             # 新增：签章/加密完成
→ bid_submitted          # 重命名：取代 delivered
→ bid_opened             # 新增：已开标
→ bid_evaluating         # 新增：评标中
→ bid_clarification      # 新增：澄清答疑（按需）
→ awarded / not_awarded / abandoned   # 新增：结果分支
→ contract_signed        # 新增：合同签订（中标后）
→ archived
```

### 3.2 招标文件解析（spec §2.2 / plan Phase 3）

**电力施工招标文件解析必须显式抽取：**

- **投标人须知前附表**（最关键，所有阈值字段）
- **评标办法前附表**（评分细则）
- **资格审查表 / 资格审查办法**（资质门槛、业绩门槛、人员门槛）
- **投标函 / 投标函附录** 模板要求
- **通用合同条款 vs 专用合同条款** 差异（专用条款的修改才是关键）
- **技术规范书 / 技术规格** 与图纸
- **答疑文件 / 补遗文件 / 澄清文件**：与原招标文件的覆盖与合并关系（缺漏即废标）— **plan Phase 3 应新增任务"招标答疑/补遗合并与变更追踪"**

**电力业务特有抽取项:**

- 电压等级、容量、规模
- 是否要求安全生产许可证及等级
- 是否要求电力设施承装/承修/承试许可证
- 是否要求特种作业证（高压电工、登高架设、电焊工等）
- 是否要求工作票/操作票管理体系
- 是否要求带电作业资质
- 类似业绩的电压等级匹配要求（如"近 5 年承担过 2 项 110kV 及以上变电站工程"）

### 3.3 约束分类（spec §2.3 / plan Phase 3）

当前分类不足，建议**约束 type 维度**增加以下类别（与原"业务/技术/评分/废标"并列）：

- `substantive_response`：实质性响应条款（不响应即废标，单独标签 + 高优先级）
- `rejection_clause`：废标条款（已有但建议提升为独立筛选维度）
- `disqualification_pre`：资格预审硬条件
- `disqualification_post`：资格后审硬条件
- `seal_signature`：签章/盖章/正副本/密封要求
- `validity`：投标有效期、保证金时效、证书有效期
- `pricing_constraint`：报价相关约束（占位字段，首期不消费但保留）

### 3.4 模板包选择因子（spec §2.4 / plan Phase 2）

选择维度需扩展为**多因子加权**：

| 因子 | 权重建议 | 说明 |
|---|---|---|
| `business_line` | 最高 | 4 类业务线模板差异显著，必须精确匹配 |
| `voltage_level` | 高 | 模板内的方案章节按电压等级分支 |
| `evaluation_method` | 中 | 综合评估法 vs 最低价法 vs 合理低价法的章节侧重不同 |
| `qualification_review_type` | 中 | 资格预审 vs 后审的资格审查册组织不同 |
| `employer_type` | 中 | 国网/南网模板风格、章节顺序差异 |
| `is_consortium_allowed` | 低 | 触发联合体协议章节 |
| `is_subcontract_allowed` | 低 | 触发拟分包项目情况表章节 |

无匹配时应**明确提示并要求人工选择**，不得静默回退（spec §2.4 已有此要求，plan Phase 2 测试需覆盖）。

### 3.5 资格商务部分（spec §3 重写）

> spec §3 当前标题为"商务标编写建议"。建议改为 **"资格商务部分编写建议"**，并明确"投标报价不在 tender 范围内，由外部文件挂载"。

**3.5.1 资质证照（电力特化）**

公司库需结构化存储：

- 营业执照（统一社会信用代码、注册资本、经营范围）
- **建筑业企业资质**（输变电工程专业承包等级、机电工程总承包等级、施工总承包等级）
- **电力设施承装/承修/承试许可证**（一级—五级）
- **安全生产许可证**（电力工程类）
- **质量/环境/职业健康安全管理体系认证**（ISO9001/14001/45001）
- 信用评级、AAA 信用证书
- 工会、党建、保密等附加资质

每张证照需存：编号、发证机关、发证日期、有效期、扫描件、关联许可范围。系统应**主动校验在投标有效期内不会过期**。

**3.5.2 人员资质（电力特化）**

人员库的关键校验点：

- 注册建造师（机电 / 电力专业，一级/二级）
- 安全工程师、注册安全工程师
- **安全员 ABC 证**（A 主要负责人、B 项目负责人、C 专职安全员）
- **特种作业证**（高压电工、低压电工、登高架设、起重、电焊工）
- 工种技能证（变电安装工、线路架设工、试验工、调试工等）
- 项目经理业绩（必须有同类项目经验）
- **社保校验**：项目经理、技术负责人、安全员等关键岗位**最近 N 个月社保必须在本单位**（电力招标常见硬性要求）

> plan Phase 5 必须显式补"人员证书与社保的关联校验"任务。

**3.5.3 业绩**

业绩四件套：**中标通知书 + 施工合同 + 竣工验收证明 + 业主出具的履约证明**，缺一不可，系统应主动检测缺件。

业绩匹配维度：

- 电压等级匹配（投标要求 110kV，业绩必须 ≥110kV）
- 业务类型匹配（变电站工程业绩不能直接用于线路工程业绩响应）
- 时间窗匹配（如"近 5 年"）
- 规模匹配（金额、容量、长度等）
- 国网/南网平台备案信息（如有）

**3.5.4 投标函与法定文件**

需在 spec §3 显式列出（当前文档未提）：

- 投标函（含报价占位，由外部填入或挂载）
- 投标函附录
- 法定代表人身份证明 / 授权委托书（注意有效期与投标有效期一致）
- 投标承诺书（廉洁、不串标、不挂靠等）
- 联合体协议（如适用）
- 拟分包项目情况表（如适用）

### 3.6 技术标章节（spec §4 重写）

> spec §4.1 章节列表偏服务/咨询类，建议**按业务线分别给出推荐章节**，由模板包承载。

**3.6.1 共性章节（4 类业务线均包含）**

- 工程概况、特点与重难点分析
- 项目组织机构与岗位职责
- 总体施工部署 / 总体实施方案
- 主要施工方案与施工方法（按业务线展开）
- 施工总进度计划及保证措施（含网络计划图）
- 资源投入计划（劳动力 / 机械设备 / 工器具 / 材料）
- 质量保证措施
- 安全生产保证措施（电力安规、工作票/操作票、反违章）
- 文明施工与环境保护措施
- 应急预案（人身触电、倒杆、火灾、设备事故、高空坠落）
- 工期、质量、安全、文明施工承诺
- 评分点逐项响应一览表
- 商务/技术偏离表（如允许偏离）

**3.6.2 业务线差异化章节**

| 业务线 | 特化章节建议 |
|---|---|
| 变电劳务及专业分包 | 班组组织与持证情况、与总包配合方案、变电设备安装/调试/试验工艺、停电配合方案、特种作业管理 |
| 10kV 项目 | 配电线路施工方案（架空/电缆）、配电箱/环网柜安装、停电计划与不停电措施、户表工程、与运维单位的接收交接 |
| 电力运维项目 | 运维管理体系、值班/巡检/缺陷管理、应急抢修响应（含响应时长承诺）、信息化平台、备品备件库、典型缺陷案例与处置 |
| 用户高低压供配电 | 一次/二次系统说明、保护与自动装置配置、调试与试验方案、并网申请与配合、用户报装流程、试运行与移交 |

模板包应按业务线分别准备，不要让 AI 在一个大模板里硬选分支。

**3.6.3 必备技术附件**

- 主要施工机械设备表
- 主要试验检测设备表
- 主要管理人员一览表
- 主要施工人员（含特种作业人员）一览表
- 拟投入工器具表
- 现场临时用电方案
- 季节性施工措施（雨季/冬季/夏季高温）

### 3.7 图表（spec §5 / plan Phase 7）

当前仅 5 类图表不足，建议**电力施工首期至少补充**：

| 图表类型 | 业务必要性 |
|---|---|
| `org_chart` 项目部组织架构图 | 已有，保留 |
| `process_flow` 通用施工流程图 | 已有，建议改名 `construction_flow` 并区分专业 |
| `schedule_gantt` 进度甘特图 | 已有，保留 |

### 3.8 审查与符合性检查（spec §6 / plan Phase 8）

**核心建议：在初步评审之前，独立增加"符合性检查（废标红线扫描）"**。这一层不与 AI 内容审查混在一起，必须确定性、可重复、能列单。

**符合性检查清单（电力施工典型）:**

- 投标文件签字盖章完整性（封面、投标函、各册首页、骑缝章）
- 法定代表人身份证明 / 授权委托书有效期（≥ 投标有效期）
- 投标保证金（金额、形式、到账截止时间）
- 资格审查硬指标（资质等级、注册资本、安全许可证、电力设施许可证）
- 项目经理 / 技术负责人 / 安全员 ABC 证持证 + 在册（社保）
- 特种作业证齐全（按招标要求）
- 业绩四件套齐全（中标通知书 + 合同 + 竣工验收 + 业主证明）
- 业绩电压等级与时间窗匹配
- 实质性响应条款逐条响应（无负偏离）
- 投标有效期满足要求
- 正本/副本/电子版份数与密封要求

输出按 P0/P1/P2/P3 分级。**P0 未关闭则禁止进入 final_layout**。

**审查矩阵新增"偏离表"产物（plan Phase 8 任务）:**

- 商务偏离表（招标商务条款 vs 投标响应）
- 技术偏离表（技术规格 vs 投标方案）
- 自动标注 实质性偏离 / 非实质性偏离
- 实质性偏离需人工逐条确认或建议放弃投标

## 4. 流程合规与生命周期补强

### 4.1 排版、签章、密封、递交（spec §7 / plan Phase 9）

需补充的合规要件：

- **正本 / 副本 / 电子版份数**（招标文件常要求"正本 1 份、副本 N 份、U 盘 N 份"）
- **纸质标书密封**：袋数、密封章位置、骑缝章、外包装标识、密封袋封皮模板
- **电子标书加密**：CA 签章 / 平台加密（首期由用户在外部完成，tender 提供导出文件 + 操作指引）
- **投标保证金缴纳凭证**作为单独附件管理
- **递交回执 / 签到记录**留存
- 投标文件正本签章页打印指引

> tender 首期不直连招投标平台，但**应导出"递交准备清单"**（清单 + 文件 + 密封说明），用户携带去现场或上传到平台。

### 4.2 开标—评标—中标段（spec §7.3 / plan Phase 10）

当前 plan Phase 10 仅记录"是否中标 + 排名 + 复盘"，建议增加：

- 开标记录（开标价格、开标时间、参与单位、开标异常情况）
- 评标过程澄清（评标委员会发出的澄清问题与公司答复）
- 中标候选人公示（公示开始时间、结束时间、是否被投诉）
- 投诉/质疑处理记录
- 中标通知书 / 未中标通知
- 合同签订时限与状态
- 履约保证金缴纳
- 公司在该业主处的"信用积分变化"（如有）

这些是后续模板优化、客户偏好沉淀、信用维护的真实数据来源。

### 4.3 风险与控制（spec §11）

补充电力施工高发风险：

- **截止时间风险:** 递交逾期即废标。系统应有倒计时、72h/24h/2h 预警，强烈建议预留至少 2 小时完成递交（电子招投标平台高峰期常拥堵）。
- **保证金风险:** 到账截止时间通常**早于**递交截止时间，需独立追踪。
- **实质性响应风险:** 实质性偏离即废标，列入审查 P0。
- **签章/盖章风险:** 缺章漏章直接废标，符合性检查必扫描。
- **资质有效期风险:** 资质证书、安全生产许可证、电力设施许可证、项目经理证书在投标有效期内不能过期。
- **社保在册风险:** 关键岗位人员近 N 个月社保必须在本单位，否则被视为挂靠。
- **答疑/补遗遗漏风险:** 招标人发布的补遗未合并进投标文件，按废标处理。

## 5. 扩展接口预留建议（为未来通用化）

虽然首期不做，但为避免未来重写，建议从一开始预留：

| 扩展方向 | 预留点 |
|---|---|
| 经济标 / 投标报价 | (1) 项目模型保留 `pricing_module_enabled` 开关；(2) 章节生成策略增加 `external_attached`；(3) 服务层注释占位 `BidPricingService`；(4) 工作流增加 `external_pricing_merge` 状态 |
| 其他工程类别（房建/市政/公路等） | (1) 模板包类型用 `industry + sub_type` 二级，首期 `industry=power`；(2) 业务线枚举可扩展；(3) 约束分类不绑定电力词汇，使用通用语义 + 子标签 |
| 招投标平台直连 | (1) 服务层注释占位 `PlatformIntegrationService`；(2) 导出层抽象出 `submission_target` 概念，首期实现 `local_zip`，未来扩展 `gw_ecp` / `csg_ebid` 等 |
| 电子保函 | 投标保证金字段 `bid_bond_form` 设计为枚举，预留 `electronic_guarantee` 值 |
| 造价软件深度集成 | 暂不设计，但导入接口预留 `external_pricing_file` 上传位 |

> 这一节建议在 spec 最后单独成章「§12 扩展接口预留」，让上下游团队明确边界与未来路径。

## 7. 待业务方确认决策项

落地前建议业务方明确：

1. 4 类业务线首期模板包来源（已有历史投标文件抽取 / 邀请专家编写 / 引用国网南网通用模板）。
2. 业务线之间是否存在交叉组合项目（如同时含 10kV + 用户配电），系统按主业务线还是允许多选。
3. 外部报价文件的格式与挂载方式（PDF / Excel / 加密标书 / 平台特定格式）。
4. 人员社保校验的数据来源（公司 HR 系统接口 / 手工上传社保证明 / 暂不校验仅提示）。
5. 业绩四件套是否首期都要求齐全，缺件是阻断还是降级提示。
6. 符合性检查的判定权限：系统给红/黄/绿，最终由谁拍板（项目经理 / 投标负责人 / 商务总监）。
7. 投标资料库与项目工作区的关系：公司库是租户级单库，还是按业务线分库。
8. 中标后开标段数据是否进入 tender 维护（手工录入 / 由其他系统对接）。

## 8. 优先级建议

如果第一阶段时间紧张，建议按以下顺序：

**P0（必须做，否则首期不可用）:**
- 业务线建模 + 模板包按业务线组织
- 项目立项字段补全（含递交截止/保证金/投标有效期/业务线/电压等级）
- 工作流状态完整化（至少到 submitted + bid_opened）
- 符合性检查（废标红线扫描）独立闸门
- 资格商务装配（资质 + 人员 + 业绩 + 法定文件）
- 技术标章节按业务线模板化
- 答疑/补遗合并

**P1（首期强烈建议）:**
- 偏离表生成
- 施工现场总平面图、网络计划图、安全/质量体系框图
- 社保在册校验、证书有效期校验
- 递交清单导出（不直连平台）

**P2（首期可选，第二阶段补）:**
- 评分对照复审（量化得分区间）
- 技术能力库（历史片段复用）
- 开标后完整数据采集

**P3（占位即可，第三阶段做）:**
- 外部报价挂载流程
- 平台直连
- 通用化扩展（其他工程类别）

## 9. 结论

两份文档作为产品蓝图方向正确、结构完整。结合「电力施工首期 + 不做经济标 + 保留扩展接口」的定位修订后，本审核报告认为可以作为 P0/P1 阶段的实施依据。

**最关键的三处必改:**

1. **业务线显式建模**（4 类业务线驱动模板、章节、资质、人员、业绩、评分映射）。
2. **符合性检查（废标红线扫描）独立成层**，作为 P0 闸门，与 AI 内容审查解耦。
3. **工作流生命周期补全到中标 + 合同签订**，否则 Phase 10 复盘数据不完整、模板优化飞轮无法转起来。

**最关键的两处接口预留:**

1. 数据模型与服务层为「经济标 / 报价」与「平台直连」预留显式扩展点。
2. 模板包类型采用 `industry + sub_type` 二级结构，避免未来扩展到房建/市政时的大重构。
