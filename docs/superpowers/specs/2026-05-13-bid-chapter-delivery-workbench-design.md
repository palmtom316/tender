# Bid Chapter Delivery Workbench Design

## Goal

将“投标文件模板/章节编辑”从内部字段配置界面，改成投标工程师能直接交付的章节工作台：资料编排型章节用“模板预览 + 资料位绑定”，AI 正文型章节用“写作要求 + 生成正文”，AI 图表型章节用“图表任务卡 + 数据/结构确认 + 预览 + 锁定插入”。

## Problem

当前模板页面右侧暴露“模板项字段面板 / 绑定编辑器 / 渲染预检 / 映射建议 / 上下文预览”五张卡片。它们反映的是系统内部模型：binding、output key、source filters JSON、field mappings、context、resolved bindings。投标工程师的真实任务不是维护这些概念，而是判断：

1. 这份标书章节能否交付。
2. 哪些章节缺资料或缺确认。
3. 缺什么资料，去哪里选，选完是否可导出。
4. AI 生成内容和图表是否满足评分点与招标约束。

根因不是文案不够易懂，而是“模板管理员/开发者配置模型”被放到了“投标工程师日常交付”路径上。

## Users and Modes

### Default: 投标工程师交付模式

默认进入交付模式。投标工程师看到业务语言：章节、资料位、缺失资料、写作要求、生成正文、图表任务、锁定状态、导出阻塞项。

不默认显示：output_key、source_filters JSON、target_field/source_field、Context、Resolved Bindings。

### Advanced: 模板管理员维护模式

保留现有绑定、字段映射、JSON、上下文、映射建议和预检细节，但移动到“高级维护”折叠区或维护抽屉。管理员用于建模板、调绑定、排查渲染。

## Chapter Types

### 1. 资料编排型章节

适用：不用 AI 生成正文，只需要固定模板文字 + 公司资料/人员资料/业绩/证书/附件编排的章节。

交互模型：

- 左侧或主区域：章节模板预览页面。
- 模板固定文字直接展示。
- 固定文字旁列出资料清单，也称“资料位”。
- 右侧：公司资料库、人员资料库、附件/业绩清单。
- 用户把资料绑定到资料位，而不是自由插入任意位置。

示例：

```text
第 3 章 企业资信情况

固定文字：
我公司具备承担本项目的相关资质……

资料清单：
[未完成] 企业营业执照
[已完成] 承装/承修/承试许可证
[未完成] 安全生产许可证
[已完成] 近三年类似业绩
```

右侧资料库：

```text
公司资料库
- 营业执照
- 承装承修承试许可证
- 安全生产许可证
- ISO 体系认证
- 类似业绩

人员资料库
- 项目经理
- 技术负责人
- 安全员
- 质检员
```

用户动作：

- 将“安全生产许可证”绑定到“安全生产许可证资料位”。
- 将“张三”绑定到“项目经理资料位”。
- 将“XX 配网工程业绩”绑定到“类似业绩资料位”。

约束：

- 不做任意位置自由拖放。
- 模板预先定义资料位：资料类型、是否必填、单选/多选、渲染形态、导出阻塞规则。
- 首期如果后端没有资料位模型，前端先用章节元数据和预检结果呈现“交付清单”体验，绑定动作可先走已有人员/设备选择接口或作为 UI 框架预留。

### 2. AI 正文型章节

适用：技术标第 8/9/10 章等需要根据招标要求、评分点、标准条款、公司能力和项目资料生成正文的章节。

不默认暴露 raw prompt，改为业务语言：

- 写作目标
- 必须响应
- 引用资料
- 生成风格
- 目标页数
- 高级提示词（折叠编辑）

推荐双栏：

```text
左侧：章节要求 + 引用资料 + 图表任务 + 高级提示词
右侧：生成正文 + 图表预览 + 审阅/保存
```

示例：

```text
第 8 章 施工方案及技术措施

写作目标：
围绕本项目配网工程特点，生成可执行的施工方案。

必须响应：
- 施工组织安排
- 主要工序施工方法
- 停电、带电、交叉作业措施
- 质量、安全、进度控制点

引用资料：
[已选] 招标文件技术要求
[已选] 公司施工组织经验
[已选] 类似业绩：XX 配网改造工程
[未选] 项目现场条件

生成风格：
[国网技术标] [正式严谨] [避免空话套话]

高级提示词：展开编辑
```

### 3. AI 图表型章节元素

图表不是装饰，而是证据化表达工具。图表交互必须回答：

1. 图表响应哪个评审要点。
2. 数据或结构从哪里来。
3. 投标工程师能否校核。
4. 是否已人工确认并锁定。
5. 是否能稳定导出到 DOCX/PDF。

不要把图表设计成“AI 随便生成一张图片”。应设计成“图表任务卡”。

## Chart Types and Interaction

### A. 结构化数据图表

示例：

- 进度计划表
- 工期横道图
- 人员投入计划表
- 机械设备投入表
- 质量检查频次表
- 安全风险清单
- 材料进场计划

交互：

```text
图表名称：施工进度计划横道图
用途：响应工期及进度保证措施
数据来源：
- 招标工期：90 日历天
- 关键节点：开工、基础施工、电缆敷设、设备安装、调试验收
- 公司类似项目进度经验

用户可编辑：
任务名称 | 开始时间 | 持续时间 | 责任班组 | 前置任务

操作：
[重新生成图表] [编辑数据] [锁定图表] [插入正文]
```

原则：用户编辑数据表，AI 生成初稿，系统稳定渲染图表。

### B. 逻辑结构图 / 流程图

示例：

- 施工组织机构图
- 质量保证体系图
- 安全管理体系图
- 应急处置流程图
- 施工工序流程图
- 缺陷处理闭环图
- 文明施工管理流程图

交互：

```text
图表名称：质量保证体系图
图表类型：组织/体系结构图

节点清单：
- 项目经理
- 技术负责人
- 质检员
- 施工班组
- 材料员
- 资料员

关系：
项目经理 -> 技术负责人
技术负责人 -> 质检员
质检员 -> 施工班组

操作：
[增删节点] [调整层级] [重新排版] [锁定图表] [插入正文]
```

原则：AI 生成初稿，最终图由系统模板化图形组件或 SVG/Mermaid 渲染，不依赖不可控 AI 图片。

### C. 示意图 / 方案图

示例：

- 施工现场布置示意图
- 电缆敷设路径示意
- 作业区围挡布置图
- 交通疏导示意
- 带电作业安全距离示意

首期不做 AI 自由绘图。推荐交互：

```text
上传底图 / 选择标准示意图模板
↓
AI 根据文字要求标注重点
↓
用户确认标注
↓
插入正文
```

首期如果无底图或模板，则提供“暂不生成，仅保留文字说明”。

## Recommended Technical Chapter Organization

### 第 8 章：施工方案及技术措施

适合图表：

- 施工流程图
- 关键工序控制表
- 施工机械设备投入表
- 施工组织机构图
- 主要施工方法清单

交互重点：从施工方案结构和项目类型生成，用户确认工序、设备、人员。

### 第 9 章：进度计划及保证措施

适合图表：

- 总进度横道图
- 关键节点计划表
- 劳动力投入曲线/表
- 材料设备进场计划表
- 进度风险与应对表

交互重点：必须基于工期、开竣工时间、关键节点。用户先确认数据表，再生成图。

### 第 10 章：质量、安全、绿色施工保障

适合图表：

- 质量保证体系图
- 安全管理体系图
- 危险源辨识与控制表
- 应急响应流程图
- 文明施工/绿色施工措施表
- 质量检查闭环流程图

交互重点：模板化流程图和表格优先，不做自由 AI 图片。

## Page Structure

### For template/database page

默认：模板交付检查。

```text
[交付状态]
当前模板：需处理 3 项后可导出
按钮：只看待处理项 / 重新检查 / 导出

[待处理清单]
1. 资质证书汇总表
   问题：缺少资质证书资料来源
   建议：从“公司资质证书库”取数
   操作：选择资料来源 / 采用建议 / 交给管理员

[当前项详情]
这项会生成什么：
- 内容类型：人员简历
- 资料来源：人员资料库 > 项目经理
- 状态：缺少安全员 B 证

[生成预览]
自然语言展示，不显示 JSON。

[高级维护]
折叠：绑定规则、字段映射、渲染上下文、Resolved Bindings
```

### For authoring/editor page

默认：章节交付工作台。

```text
左侧：章节导航
- 章节号/标题
- 类型：资料编排 / AI 正文 / 图表任务
- 状态：待补资料 / 待生成 / 待审阅 / 已锁定

中间：章节预览
- 资料编排型：固定文字 + 资料位清单 + 绑定状态
- AI 正文型：写作要求 + 正文编辑预览
- 图表型：图表任务卡 + 图表占位/预览

右侧：资料/依据/图表库
- 公司资料库
- 人员资料库
- 图表任务
- 引用依据和风险提示
```

## Initial Implementation Scope

首期目标是建立正确交互骨架，尽量复用现有接口：

1. `TemplateFieldWorkbench` 默认显示“交付模式”，原五张技术卡片移到“高级维护”折叠区。
2. `EditorContent` 将现有章节编辑重组为章节交付工作台：章节导航、章节控制/资料位/图表任务、正文预览。
3. 现有图表生成从“图表类型下拉 + 生成”升级为“图表任务卡”：用途、来源、缺失项、状态、生成/审批/插入动作。
4. 不新增后端资料位模型；首期使用现有 outline、draft、technical context、chart assets、business assembly、project personnel/equipment/company evidence 接口做展示和已有动作。
5. 所有原有能力仍可访问，不删除高级维护能力。

## Acceptance Criteria

1. 投标工程师默认看不到 JSON、output_key、target_field/source_field、Context、Resolved Bindings。
2. 模板页面默认显示“交付状态 + 待处理清单 + 当前项生成预览 + 高级维护折叠”。
3. 章节页面能区分资料编排型章节和 AI 正文型章节，并用不同业务文案展示。
4. 技术章节显示写作要求、引用资料计数、目标页数和图表任务。
5. 图表资产以任务卡展示，至少包含：图表名称、类型、用途、来源/占位符、状态、生成/审批/插入动作。
6. 已有章节生成、技术生成、目标页数保存、图表生成、图表审批、图表插入、草稿保存功能继续工作。
7. Vitest 覆盖新增关键交互。

## Non-goals

首期不做：

- 后端资料位绑定模型重构。
- 真正拖拽交互。
- AI 自由生成现场示意图。
- 图表数据表在线编辑完整实现。
- DOCX/PDF 渲染引擎重写。
- 权限系统区分投标工程师和模板管理员。
