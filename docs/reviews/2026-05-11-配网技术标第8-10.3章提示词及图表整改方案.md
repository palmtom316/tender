# 国网配网工程技术标第 8/9/10.1/10.2/10.3 章提示词与图表整改方案

- 制定日期：2026-05-11
- 制定依据：`docs/reviews/2026-05-11-配网技术标第8-10.3章提示词及图表审查报告.md`
- 适用范围：tender 系统配网工程技术标量产链路
- 文档定位：本方案是审查报告 § 六《整改建议》和 § 七《放行结论》的**工程化执行计划**，明确"做什么、谁来做、何时完成、如何验收"。所有任务编号沿用审查报告中的问题编号（Q-x-y / CR-x），保证整改可追溯。

## 一、整改目标与原则

### 1.1 整改目标

| 维度 | 当前状态 | 目标状态 | 关键 KPI |
| --- | --- | --- | --- |
| 提示词工程化 | 5 份提示词体系成熟，存在 ~20 处细节增补点 | 5 份提示词无 Q 级遗留问题 | Q-8-x / Q-9-x / Q-10.x-y 问题清单全部销项 |
| 图表渲染稳健性 | flow 渲染垂直链、甘特图无依赖与刻度、暗标未脱敏 | sidecar 不可用时回退渲染仍可读，暗标项目不外泄 | CR-1 ~ CR-5 全部修复 |
| 图表配置完整度 | `required_charts` 共 10 个占位，落地比 ~14% | 落地比 ≥ 70%，关键评标点全部有图 | 见 § 三. P1.5 配置补齐清单 |
| 跨章节一致性 | 质量/安全/进度边界清晰，绿色/地域/数字化分工不显式 | 5 主题跨章节分工显式声明 | 跨章节归属矩阵全绿 |
| 暗标合规 | 单点风险：AI 网关 spec 未脱敏 | 暗标项目下 spec 上行前完成关键字段移除 | 暗标项目导出审计 0 命中 |

### 1.2 整改原则

1. **优先级驱动**：P0（影响评标可用性）→ P1（影响完整度）→ P2（影响体验），不允许 P0 未完成即放行。
2. **可追溯**：每个整改项关联审查报告原问题编号，commit/PR 标题必须带 `[REM-<编号>]`。
3. **不改提示词哲学**：5 份提示词的"约束驱动、来源溯源、暗标控制、跨节边界"四大基础不动，只做细节增补和能力承接。
4. **代码与提示词对齐**：任何"建议图表"提到的能力，要么在代码侧承接，要么从提示词中下沉为"用 Markdown 表格表达"。**两边对不齐的状态不允许长期存在**。
5. **小步快跑**：拆分到提交粒度，每个 P0/P1 子项独立 PR，独立验收。
6. **测试先行**：图表渲染相关修改必须先补单元测试（覆盖错误拓扑、空依赖、暗标关键词），再做实现。

### 1.3 不在本方案范围

- 第 6 章（项目团队情况）、第 12 章（技术评分支撑材料）、第 13 章（技术规范响应）等其他章节的提示词工程化；
- 商务标章节；
- AI 模型选型与 fallback 策略调整；
- 招标文件解析、约束抽取等上游链路。

### REM-C-1：修复 flow 渲染拓扑（CR-1）

- **问题**：`renderers.py::_render_flow_svg` 忽略 `spec.edges`，所有节点纵向排列。
- **目标**：fallback 渲染能正确表达树/DAG 拓扑；Mermaid sidecar 不可用且 fallback 仍不可用时**拒绝渲染**并标记 `needs_review`。
- **改动文件**：
  - `backend/tender_backend/services/chart_service/renderers.py`
  - `backend/tender_backend/services/chart_service/specs.py`（如需新增字段）
  - `backend/tests/unit/test_chart_rendering_and_injector.py`
- **实施步骤**：
  1. 新增 `_layout_dag(nodes, edges, direction)` 函数：基于简化 Sugiyama / Reingold-Tilford 算法计算节点坐标（按 direction 处理 TB/LR）；
  2. 重写 `_render_flow_svg`：从 layout 结果绘制节点矩形 + edges 折线箭头；
  3. `render_chart_spec` 在 sidecar 失败 + fallback 输出节点数 ≤ 1 或 edges 全无连通时，抛出 `ChartRenderError`；
  4. `ChartGenerationService.create_or_update` 捕获 `ChartRenderError` 后将 status 设为 `needs_review`，metadata_json 记录失败原因；
  5. 单测：构造 3 类典型拓扑（线性 / 树 / 单源多汇 DAG），断言 fallback SVG 包含全部 edges 元素。
- **验收标准**：
  - 单元测试覆盖率 ≥ 90%；
  - 用第 8.5、10.1.3、10.2.2 三个 quality_system / safety_system 体系图做 e2e 验证，sidecar 关闭情况下输出 SVG 含分支结构；
  - 评标专家盲评：3 个项目的体系图"组织层级清晰"判定通过。
- **回滚预案**：渲染失败时不输出错误图，而是 placeholder 保持 `{{chart:*}}` 并在 review_engine 中标红，由人工绘图替代。

- **实施步骤**：
  1. 计算工期跨度 `total_days`，按以下规则选刻度：

### 9.2 集成测试

### 9.4 回归基线
