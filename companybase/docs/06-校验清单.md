# 校验清单

## P0 必须修复

- `unique_key` 为空或重复。
- `company_key` 在 `公司主体` 中不存在。
- `owner_type` 不属于系统允许值。
- `owner_unique_key` 找不到对应主数据。
- `file_relative_path` 为空、文件不存在或路径逃逸到 `companybase/` 外。
- 附件扩展名不在 PDF/图片白名单内。
- 日期格式不是 `YYYY-MM-DD`。
- JSON 字段不是合法 JSON object。
- `company_asset.asset_type` 不是 `vehicle/machine/tool/safety`。
- `company_asset.ownership` 不是 `self/leased/third_party`。
- `company_asset.status` 不是 `active/maintenance/retired`。

## P1 应修复

- 企业资质缺证书编号或有效期。
- 人员缺岗位、专业、关键证书附件。
- 设备缺规格型号、编号或检定/保险到期日。
- 业绩缺合同附件或验收证明。
- 附件未填写 `asset_domain/asset_category/asset_type`。
- 证书、设备检定、保险、人员证书已过期。
- 证书或设备检定 90 天内到期。
- 暗标敏感附件未标记 `is_blind_sensitive=TRUE`。

## P2 可优化

- 文件名不清晰。
- `metadata_json` 缺导入来源或备注。
- 业绩摘要过短，无法支持技术标生成。
- 制度能力未关联适用章节。
- 资产数量过于笼统，缺少适用工序说明。

## 上线导入前检查

- Excel 模板版本和导入脚本版本一致。
- 附件 hash 清单已生成。
- 开发库 dry-run 无 P0。
- 上线库 dry-run 无 P0。
- 导入报告记录数与预期一致。
- 抽样下载附件可以打开。
- 暗标敏感字段能被生成上下文脱敏。
