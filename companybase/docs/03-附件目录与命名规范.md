# 附件目录与命名规范

## 支持格式

当前系统证据附件上传接口支持：

- PDF：`.pdf`
- 图片：`.png`、`.jpg`、`.jpeg`、`.bmp`、`.gif`、`.tif`、`.tiff`

系统会校验扩展名、文件头、content-type 和大小限制。不要把 Word、Excel、压缩包直接作为 `evidence_asset` 上传；如确需保留，可先转 PDF 或在后续扩展导入器支持。

## 目录结构

```text
files/
  company/        # 公司主体资料、营业执照、体系认证、法人授权等
  qualification/  # 企业资质、许可证、认证证书
  personnel/      # 人员证书、社保、劳动合同、培训证明
  assets/         # 车辆、机械、工器具、安全器具附件
  performance/    # 业绩合同、验收证明、评价资料
  finance/        # 审计报告、财务报表、纳税证明
  methods/        # 制度、工法、应急预案、数字化能力证明
```

## 文件命名

建议命名公式：

```text
业务对象_附件类型_有效期或年份.pdf
```

示例：

```text
安全生产许可证_2026-12-31.pdf
承装修试许可证_三级_2027-08-15.pdf
项目经理001_一级建造师证_2028-06-30.pdf
项目经理001_安全B证_2027-12-31.pdf
绝缘电阻测试仪001_检定证书_2026-05-31.pdf
2025配网改造项目_合同关键页.pdf
2025配网改造项目_竣工验收证明.pdf
质量管理制度_现行版.pdf
```

## 核心附件槽建议

### 企业资质

- `certificate_scan`：证书扫描件。
- `change_record`：变更/延期证明。
- `verification_screenshot`：官网查询或核验截图。

### 人员资料

- `id_certificate`：身份证明，通常暗标敏感。
- `professional_certificate`：建造师、职称、岗位证。
- `safety_certificate`：安全 B/C 证等。
- `social_security`：社保证明。
- `training_record`：培训证明。

### 公司资产

- 车辆：`driving_license`、`annual_inspection`、`insurance`。
- 机械：`factory_certificate`、`inspection_report`、`purchase_or_lease`。
- 工器具：`inspection_report`、`factory_certificate`。
- 安全器具：`factory_certificate`、`periodic_test_report`。

### 项目业绩

- `contract`：合同关键页。
- `acceptance_report`：验收证明。
- `performance_evaluation`：业主评价或履约评价。
- `invoice_or_settlement`：发票/结算证明，如技术标暗标通常不使用。

## 文件安全

- 不要把附件放在 `companybase/` 以外再用绝对路径引用。
- 不要上传含宏文件或未知来源文件。
- 含身份证、手机号、联系人、公司地址、Logo 的图片和 PDF 应在 `附件索引.is_blind_sensitive` 标记 TRUE。
- 对外备份时应使用加密压缩包或公司文件管理系统权限控制。
