# 备份与版本管理

## 备份目标

公司资料包含证书、人员、业绩、设备和扫描件，是长期资产。备份目标不是只备份数据库，而是备份可迁移的数据包：

- Excel/CSV 结构化数据。
- PDF/图片附件。
- 导入报告和校验报告。
- 文件 hash 清单。

## 推荐版本号

使用日期和批次命名：

```text
companybase_2026-05-13_v1
companybase_2026-05-13_v2
companybase_2026-06-01_monthly
```

## 备份命令

本目录提供脚本：

```bash
bash companybase/tools/backup_companybase.sh
```

默认输出：

```text
companybase/backups/companybase_YYYYMMDD_HHMMSS.tar.gz
companybase/backups/companybase_YYYYMMDD_HHMMSS.sha256
```

## 备份频率

- 每次大批量录入后立即备份。
- 每次上线导入前备份。
- 每次证书、人员、设备、业绩批量更新后备份。
- 至少每月做一次月度备份。

## 备份内容

必须包含：

```text
README.md
docs/
templates/
imports/
files/
exports/
```

可选包含：

```text
tools/
```

不建议包含：

```text
backups/
```

避免备份包套备份包无限膨胀。

## 恢复流程

1. 解压备份包到一个新目录。
2. 校验 `.sha256`。
3. 运行 `tools/validate_companybase.py`。
4. 用 `--dry-run` 导入目标数据库。
5. 确认报告无 P0 错误。
6. 正式导入。

## 安全要求

资料包可能包含身份证、手机号、合同、证书编号、公司地址和商业信息，应按敏感资料管理：

- 不提交真实附件到公开仓库。
- 不通过普通聊天工具传未加密压缩包。
- 对外共享前可使用脱敏副本。
- 上线环境备份应放在受控存储，并限制访问权限。

## 数据库备份与资料包备份的关系

数据库备份用于恢复运行状态；资料包备份用于跨环境重建主数据。两者都需要，但不能互相替代。

推荐做法：

- 开发/测试/上线数据库可各自备份。
- 公司资料以 `companybase` 作为单一事实来源。
- 上线迁移时从 `companybase` 重新导入，而不是从开发库复制数据。
