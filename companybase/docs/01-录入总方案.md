# 公司资料一次录入多次复用方案

## 目标

建立一套可迁移、可校验、可反复导入的公司资料基座，使 tender 系统在生成技术标、商务标、资格证明、设备表、人员表和证据包时都从同一套主数据取数。

这套方案解决三个问题：

1. 开发系统和上线系统不是同一个数据库，避免重复手工录入。
2. 结构化字段和 PDF/图片证据附件分开管理，但能稳定关联。
3. 后续招标文件端到端验收时，AI 章节能拿到足够的公司、资质、人员、设备和业绩上下文。

## 数据分层

### 1. 主体层

`公司主体` sheet 对应系统 `library_company`，用于区分投标主体、联合体成员、分公司或常用供应商。

关键字段：

- `company_key`：稳定唯一键，例如 `main_company`。
- `company_name`：公司全称。
- `company_type`：`bidder`、`branch`、`partner`、`supplier`。

### 2. 结构化主数据层

结构化数据对应系统业务表：

- `公司资料`：公司基础信息、注册地址、统一社会信用代码、经营范围。
- `企业资质`：施工资质、安全生产许可证、承装修试许可证、体系认证。
- `人员资料`：项目经理、技术负责人、安全负责人、质量负责人、资料员、特种作业人员等。
- `公司资产`：车辆、机械、工器具、安全设施设备。
- `项目业绩`：合同业绩、验收证明、评价资料。
- `财务报表`：年度资产负债、利润、现金流或摘要指标。
- `制度能力`：质量、安全、绿色、应急、数字化、标准化施工等能力材料。

每行必须填写 `unique_key`，后续导入器以它做幂等更新。

### 3. 证据附件层

所有 PDF、图片扫描件统一由 `附件索引` sheet 维护，对应系统 `evidence_asset`。

附件只通过相对路径引用，例如：

```text
files/qualification/安全生产许可证.pdf
files/personnel/张三_安全B证.pdf
files/assets/绝缘电阻测试仪_检定证书.pdf
```

不要在 Excel 中粘贴图片，也不要把文件绝对路径写入模板。绝对路径会导致换电脑、换服务器后失效。

## 推荐录入顺序

1. 录入 `公司主体`。
2. 录入 `公司资料`。
3. 录入 `企业资质`。
4. 录入 `人员资料`。
5. 录入 `公司资产`。
6. 录入 `项目业绩`。
7. 录入 `财务报表` 和 `制度能力`。
8. 将 PDF/图片放入 `files/`。
9. 录入 `附件索引`。
10. 运行校验脚本。
11. 导入开发库验证。
12. 导入上线库。

## 命名规范

### unique_key

建议全部使用 ASCII 小写、数字、下划线，长期稳定不变。

示例：

```text
main_company
cert_safety_production_license
cert_power_facility_license_level3
person_project_manager_001
asset_vehicle_pickup_001
asset_tool_insulation_resistance_tester_001
perf_2025_distribution_rebuild_001
method_quality_management_system
```

### 附件文件名

附件文件名可以中文，但建议包含业务含义，不要只叫 `1.pdf`、`扫描件.jpg`。

推荐：

```text
安全生产许可证_2026-12-31.pdf
项目经理001_一级建造师证_2028-06-30.pdf
绝缘电阻测试仪001_检定证书_2026-05-31.pdf
2025配网改造项目_竣工验收证明.pdf
```

## 幂等导入规则

导入器应按以下顺序识别重复：

1. 主数据：`record_type + unique_key`。
2. 附件：`owner_type + owner_unique_key + asset_type + file_sha256`。
3. 核心附件槽：同一个 `owner_type + owner_unique_key + asset_type` 如有新文件，可新增版本或替换最新版本，由导入参数决定。

建议默认策略：

- 主数据已存在则更新。
- 相同 hash 附件已存在则跳过。
- 同类附件 hash 不同则新增一条证据资产，不覆盖旧文件。

## 暗标处理

资料库可以保存真实名称、人员姓名、证书编号和公司名称，但生成暗标技术标时必须由系统按 `is_blind_bid` 做脱敏。

建议在 `附件索引` 中维护：

- `is_blind_sensitive`：是否暗标敏感。
- `redaction_note`：脱敏说明。

例如人员姓名、身份证、手机号、企业 Logo、公司地址、联系人均应标记为敏感。

## 最小可用数据集

为支持第 8/9/10.1/10.2/10.3 章样章验收，至少录入：

- 1 条公司主体和公司资料。
- 5-10 条企业资质/认证。
- 8-15 名项目管理及关键作业人员。
- 20-40 条车辆、设备、工器具、安全器具。
- 5-10 条配网/居配/10kV 相关业绩。
- 8-12 条制度能力材料。
- 上述记录对应的核心附件扫描件。
